O teste manual é um tipo de teste de software no qual os casos de teste são executados manualmente por um testador sem o uso de ferramentas automatizadas. O objetivo do teste manual é identificar os bugs, problemas e defeitos no aplicativo de software. O teste manual de software é a técnica mais primitiva de todos os tipos de teste e ajuda a encontrar erros críticos no aplicativo de software.

Um Plano de Teste é um documento detalhado que descreve a estratégia de teste, objetivos, cronograma, estimativa, entregas e recursos necessários para realizar o teste de um produto de software. O Plano de Teste nos ajuda a determinar o esforço necessário para validar a qualidade do aplicativo em teste.

  • Test Plan: What is, How to Create (with Example)

O teste funcional é um tipo de teste de software que valida o sistema de software em relação aos requisitos/especificações funcionais. O objetivo dos testes funcionais é testar cada função do aplicativo de software fornecendo entrada apropriada e verificando a saída em relação aos requisitos funcionais.

  • What is Functional Testing? Types & Examples
  • Functional Testing : A Detailed Guide

O teste não funcional é um tipo de teste de software para testar parâmetros não funcionais, como confiabilidade, teste de carga, desempenho e responsabilidade do software. O objetivo principal do teste não funcional é testar a velocidade de leitura do sistema de software de acordo com os parâmetros não funcionais. Os parâmetros do teste não funcional nunca são testados antes do teste funcional.

Um Caso de Teste é um conjunto de ações executadas para verificar um determinado recurso ou funcionalidade do seu aplicativo de software. Um Caso de Teste contém etapas de teste, dados de teste, pré-condição e pós-condição desenvolvidos para um cenário de teste específico para verificar qualquer requisito. O caso de teste inclui variáveis ​​ou condições específicas, com as quais um engenheiro de teste pode comparar os resultados esperados e reais para determinar se um produto de software está funcionando de acordo com os requisitos do cliente. Um Cenário de Teste é definido como qualquer funcionalidade que pode ser testada. É um conjunto coletivo de casos de teste que ajuda a equipe de teste a determinar as características positivas e negativas do projeto.

Verificação em Teste de Software é um processo de verificação de documentos, projeto, código e programa para verificar se o software foi construído de acordo com os requisitos ou não. O principal objetivo do processo de verificação é garantir a qualidade da aplicação de software, design, arquitetura, etc. O processo de verificação envolve atividades como revisões, revisões e inspeção. A validação em Engenharia de Software é um mecanismo dinâmico de teste e validação se o produto de software realmente atende às necessidades exatas do cliente ou não. O processo ajuda a garantir que o software cumpra o uso desejado em um ambiente apropriado. O processo de validação envolve atividades como teste de unidade, teste de integração, teste de sistema e teste de aceitação do usuário.

O teste de automação é uma técnica de teste de software que é executada usando ferramentas especiais de software de teste automatizado para executar um conjunto de casos de teste. Pelo contrário, o teste manual é realizado por um humano sentado em frente a um computador executando cuidadosamente as etapas do teste. O teste automatizado é a aplicação de ferramentas de software para automatizar um processo manual conduzido por humanos de revisão e validação de um produto de software. A maioria dos projetos modernos de software ágil e DevOps agora incluem testes automatizados desde o início. Para apreciar plenamente o valor dos testes automatizados, no entanto, é útil entender como era a vida antes de ser amplamente adotado.

O teste de back-end é um método de teste que verifica o lado do servidor ou banco de dados de aplicativos ou software da web. O teste de back-end visa testar a camada de aplicativo ou a camada de banco de dados para garantir que o aplicativo da Web ou o software esteja livre de defeitos de banco de dados, como deadlock, corrupção de dados ou perda de dados.

A automação de front-end é uma forma de caracterizar a automação que agiliza tarefas focadas em interatividade, sites e processos atendidos. A automação de processos robóticos, ou RPA, é considerada automação no front-end ou no nível da interface do usuário (UI). Os benefícios da automação de front-end incluem a criação rápida de tarefas sem conhecimento de programação, sem necessidade de alterações em programas ou aplicativos existentes, e os indivíduos que conhecem as teclas podem facilmente criar a tarefa de automação.

Test Driven Development (TDD) é uma abordagem de desenvolvimento de software na qual os casos de teste são desenvolvidos para especificar e validar o que o código fará. Em termos simples, os casos de teste para cada funcionalidade são criados e testados primeiro e, se o teste falhar, o novo código é escrito para passar no teste e tornar o código simples e livre de erros.

O desenvolvimento orientado a testes (TDD) é uma abordagem de desenvolvimento de software em que você escreve testes primeiro e depois usa esses testes para orientar o design e o desenvolvimento de seu aplicativo de software.
